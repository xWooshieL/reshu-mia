# Правила оформления конспектов (раздел «Обучение»)

Конспект — это не учебник и не справка, а **компактная карта** темы: читаешь сверху вниз, задерживаешься на формулах, быстро понимаешь алгоритм, сразу переходишь к примерам. Стиль — пастельный, минималистичный, без визуального шума.

---

## 1. Где живут конспекты

`site/js/learn.js` → объект `window.LEARN_DATA.notes`:

```js
notes[1] = { html: '... большая HTML-строка ...' };
```

HTML попадает в `#learn-detail-body`, затем автоматически прогоняется через KaTeX. Внутри можно использовать:
- обычный HTML (`<div>`, `<p>`, `<span>`, `<svg>`, `<table>` …),
- LaTeX через `$...$` (inline) и `$$...$$` (блочный),
- любые классы из `styles.css` (см. ниже).

---

## 2. Что можно и нельзя

### Нельзя
- `\boxed{...}` — не используем вообще (KaTeX рендерит его криво, да и визуально громоздко).
- Теговые «итемизы» списком точек `•` через `<ul>` — если очень нужен перечень, используй `.note-inline-list` (см. ниже) или пиши в два предложения.
- Яркие цвета (красный, пурпурный, ярко-синий): только мягкая пастель.
- «Боксы с жирной рамкой» — `border: 2px solid #333` и подобные. Разделение делается мягким фоном секций, не рамками.
- `\begin{itemize} ... \end{itemize}` внутри LaTeX-блоков — это LaTeX, не HTML, он не отрендерится.

### Можно (и нужно)
- Пастельные секции с мягким заливочным фоном.
- Векторные SVG-схемы с пастельной палитрой (оси и основные векторы могут быть чёрными или тёмно-серыми для контраста).
- Короткие формулы-визитки с выделением ключевой части цветом акцента.
- Шаги алгоритма через пронумерованные блоки.

---

## 3. Пастельная палитра

Задана переменными в `:root` (`styles.css`). Используй **классы**, а не хардкод цветов:

| Класс | Цвет | Когда использовать |
|-------|------|---------------------|
| `.pastel-mint` | мятный | теория, определения |
| `.pastel-peach` | персиковый | интуиция, геометрический смысл |
| `.pastel-lavender` | лавандовый | алгоритм шагов |
| `.pastel-sky` | небесный | формулы и ключевые тождества |
| `.pastel-rose` | розовый | подводные камни, типичные ошибки |
| `.pastel-sage` | шалфей | примеры, разборы задач |

Каждый класс даёт ровную мягкую заливку + тонкий левый бордер акцентного тона + аккуратный padding. Никакого `border-radius > 12px`, никакой тени.

---

## 4. Структура конспекта (каноническая)

Порядок секций подбирай по смыслу, но ядро одинаковое:

1. **Краткая идея** (`.pastel-peach`) — 2–3 предложения, зачем это в экзамене.
2. **Определения и формулы** (`.pastel-mint` + `.pastel-sky`) — только то, что реально нужно.
3. **Визуализация** — одна чёткая SVG-схема.
4. **Свойства** — короткий справочник (`.note-formula-list`).
5. **Алгоритм** (`.pastel-lavender`) — пронумерованные `.note-step`.
6. **Подводные камни** (`.pastel-rose`) — перечисление типичных ошибок.
7. **Разбор задач** (`.pastel-sage`) — 2–3 задачи из банка, с подсветкой ключевых шагов.

## 5. Ключевые CSS-компоненты

### 5.1. Секция
```html
<section class="note-section pastel-mint">
  <h3 class="note-h3">Определение</h3>
  <p>Скалярное произведение ...</p>
</section>
```

### 5.2. Формула-визитка (выделенная)
```html
<div class="note-formula">
  $$\cos\varphi = \frac{\langle \vec a, \vec b\rangle}{\|\vec a\|\,\|\vec b\|}$$
</div>
```

### 5.3. Список формул (компактный, без точек-буллетов)
```html
<div class="note-formula-list">
  <div><span class="note-label">линейность:</span>  $\langle \alpha\vec a + \beta\vec b, \vec c\rangle = \alpha\langle \vec a,\vec c\rangle + \beta\langle \vec b,\vec c\rangle$</div>
  <div><span class="note-label">симметрия:</span> $\langle \vec a, \vec b\rangle = \langle \vec b, \vec a\rangle$</div>
</div>
```

### 5.4. Шаг алгоритма
```html
<ol class="note-steps">
  <li class="note-step"><strong>1.</strong> Вычисляем $\langle \vec a, \vec b\rangle$ покоординатно.</li>
  <li class="note-step"><strong>2.</strong> Находим нормы $\|\vec a\|, \|\vec b\|$.</li>
</ol>
```

### 5.5. Инлайн-перечисление в строку (когда список уместен, но хочется компактно)
```html
<span class="note-inline-list">
  <span>билинейность</span>
  <span>симметрия</span>
  <span>положительная определённость</span>
</span>
```

### 5.6. Пример / разбор
```html
<section class="note-example pastel-sage">
  <div class="note-example__label">Пример 1 · Демо экзамена 2025</div>
  <p><strong>Условие.</strong> ...</p>
  <p><strong>Решение.</strong> ...</p>
  <p class="note-answer">Ответ: $\sqrt{22}$</p>
</section>
```

### 5.7. Подводный камень
```html
<div class="note-pitfall">
  <strong>⚠</strong> Не путай $\|\vec a - \vec b\|^2$ с $\|\vec a\|^2 - \|\vec b\|^2$ — появляется ещё и удвоенное скалярное произведение.
</div>
```

## 6. SVG-графики

### Правила
- **Оси** — чёрные или тёмно-серые (`#1a1a1a` или `#333`), `stroke-width: 1`.
- **Векторы, кривые, точки** — из пастельной палитры. Желательно с мягкой тенью `filter: drop-shadow(...)` или полупрозрачной обводкой.
- **Заливки** — полупрозрачные (`fill-opacity: 0.35–0.5`), чтобы за ними читался фон.
- **Подписи** — `font-family: "Inter", sans-serif`, размер 12–14px, цвет `#333`.
- **Размер SVG** — не больше 520px в ширину, `viewBox` обязателен для масштабирования.
- **Центр координат** — обязательно помечаем `O`, стрелочки на концах осей.

### Шаблон
```html
<svg class="note-svg" viewBox="0 0 520 360" xmlns="http://www.w3.org/2000/svg">
  <!-- оси -->
  <line x1="40"  y1="180" x2="500" y2="180" stroke="#1a1a1a" stroke-width="1"/>
  <line x1="260" y1="340" x2="260" y2="20"  stroke="#1a1a1a" stroke-width="1"/>
  <!-- стрелки осей -->
  <polygon points="500,180 492,176 492,184" fill="#1a1a1a"/>
  <polygon points="260,20  256,28  264,28"  fill="#1a1a1a"/>

  <!-- вектор a (мятный) -->
  <line x1="260" y1="180" x2="380" y2="80" stroke="#6aab9c" stroke-width="3"
        marker-end="url(#arrow-mint)"/>
  <text x="325" y="115" fill="#4d8177" font-size="14">$\vec a$</text>
</svg>
```

### Палитра для графиков (чистые значения)
- мятный (`#6aab9c` контур, `#c8e6d9` заливка),
- персиковый (`#e89f85` / `#fbd9c6`),
- лавандовый (`#9882c3` / `#dcceee`),
- небесный (`#6ba3d6` / `#cfe3f3`),
- шалфей (`#8ea878` / `#d6e3c4`).

## 7. Формулы (KaTeX)

- Inline — `$...$`. Блок — `$$...$$` (центрируется автоматически через `.note-formula`).
- Векторы — `\vec{a}`, `\vec{b}`. Не `\bar{a}`, не жирное.
- Скалярное произведение — `\langle \vec a, \vec b\rangle`. Не точка `\cdot`.
- Норма — `\|\vec a\|`. Не `|\vec a|`.
- Дроби в примерах — через `\dfrac` (а не `\frac`), чтобы не «сжимало».
- Длинные выводы — с переносами через `\\` внутри `aligned` / `array`:

```latex
$$\begin{aligned}
\|\vec a - \vec b\|^2 &= \langle \vec a - \vec b,\; \vec a - \vec b\rangle \\
 &= \|\vec a\|^2 - 2\langle \vec a, \vec b\rangle + \|\vec b\|^2.
\end{aligned}$$
```

---

## 8. Тон

- Пиши **на «ты»** и **коротко**.
- Не пересказывай учебник: объясняй только то, что реально применяется.
- Не добавляй эмодзи (кроме одного `⚠` в `.note-pitfall`).
- Не пиши «как мы видим», «очевидно» — просто показываем.
- В разборах задач всегда указывай **источник** (откуда задача) в `.note-example__label`.

---

## 9. Проверка перед коммитом

- [ ] Все `$` и `$$` парные; KaTeX не выдаёт ошибок в консоли.
- [ ] SVG помещается в ширину контейнера на мобильной версии.
- [ ] Использованы пастельные классы, а не хардкод цветов.
- [ ] Нет `\boxed`, `\begin{itemize}`, нет буллет-списков с жирными рамками.
- [ ] Есть хотя бы одна секция алгоритма и 2 разобранные задачи.
